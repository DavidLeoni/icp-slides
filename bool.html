Conversioni

  • Abbiamo visto che i possibili valori per booleani sono True e False
  • Ma booleani possono essere anche convertiti ad interi, e vice versa!
  • Come sono gestiti i valori di verità quando si effettuano queste conversioni? Prova a scoprirlo
  • Ogni intero è valutato a True, eccetto per 0
  • I valori di verità True e False si comportano rispettivamente come gli interi 1 e 0

  • Per questioni di efficienza, se durante la valutazione di un’espressione booleana Python scopre che il risultato possibile può essere solo uno, allora evita di calcolare le espressioni seguenti.

Congiungere operazioni

Dato un paio di quantità x, y = 7, 5

Come potremmo scrivere una espressione booleana per chiederci se x e y sono entrambe maggiori di zero?

Provaci!

Attenzione! La scrittura x and y > 0 NON è corretta

Perché?

Apparentemente su alcuni input sembra funzionare...

Ma funziona proprio su tutti?

Qualcosa non va...

Python ha implicitamente considerato l'espressione come se fosse con queste parentesi

(x) and (y > 0)

Quindi Python sta rilevando il valore logico dell'intero -3, che essendo diverso da 0 viene considerato True

Se invece provassimo a mettere noi le parentesi: (x and y) > 0?

Ancora non funziona... prova a pensare bene a tutti i passi che Python farà per calcolare l'espressione

Vai giù per la soluzione!

Sia x che y vengono considerate singole espressioni booleane, quindi Python internamente ridurrà l'espressione nei seguenti passi:

  1. (True and True) > 0
  2. True > 0
  3. 1 > 0
  4. True

Quindi qual è la soluzione corretta? x > 0 and y > 0

Sicuro di volere la soluzione?

Soluzione 'lunga'

Un primo problema da affrontare potrebbe essere il fatto che la lancetta può aver girato anche più di 360°. Per risolvere questo primo problema, possiamo direttamente usare il modulo: n % 360 che ci darà sempre un numero compreso tra 0 e 359 inclusi.

Già così potremmo risolvere abbastanza semplicemente l'esercizio con tanti and e or, per esempio:

(n % 360 >= 90 and n % 360 < 120) or (n % 360 >= 210 and n % 360 < 240) or (n % 360 >= 330 and n % 360 < 360)

Soluzione 'breve'

In alternativa, possiamo anche usare una soluzione breve, osservando il fatto che se consideri sequenze di 4 spicchi di 30° l'uno, ogni sequenza occupa 120° e a noi interessa sapere se la lancetta è nell'ultimo spicchio, tra 90° e 120°.

Quindi possiamo usare creativamente il modulo per 'accorciare' la plancia e ignorare tutti i gradi dopo il 120esimo, e infine guardariamo se n sta nell'ultimo spicchio:

n % 120 > 90
